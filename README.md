# 高风险客户尽调表生成器

一套面向手机填写场景的静态 PWA 工具，用于录入高风险客户尽职调查资料，并生成与原始格式保持一致的 PNG 图片表格。

在线地址：[https://gzchenzhe.github.io/wps/](https://gzchenzhe.github.io/wps/)

## 功能

- 三页分组填写：客户、贷款、完成，减少手机端长页面滚动。
- 页面固定为设备视口比例，禁止双指、双击和输入框聚焦缩放，保持接近原生 App 的操作体验。
- 根据表单内容自动生成正式 PNG 尽调表图片，不含水印。
- 保持原表格的固定结构和样式；长内容会在对应单元格内自动换行。
- 客户经理可直接手写签字，并可切换到横屏全屏扩大书写范围；保存后自动锁定并可在后续表单中复用。
- 正式图片保留客户经理签字，移除主管签字；亲见日期显示在原主管签字位置。
- 支持点击保存后直接调用手机系统文件分享面板；不支持分享时会按旧版逻辑直接触发浏览器下载/预览。
- 支持一键读取剪贴板中的客户资料，自动识别姓名、电话、单位／公司名称、居住地址，确认后填入表单。
- 支持清理贷款文字，自动删除指定的附加费用贷款说明，并允许编辑、复制清理结果。
- 可选接入腾讯云身份证 OCR，上传身份证正面图片后识别姓名、身份证号和性别，确认后填入表单。

## 手机使用

### 添加到 iPhone 主屏幕

1. 用 Safari 打开在线地址。
2. 点击底部分享按钮。
3. 选择“添加到主屏幕”。
4. 以后可从手机桌面直接打开。

### 页面说明

- `客户`：机构信息、客户信息、单位类型和居住地址。
- `贷款`：进件时间、亲见日期、贷款资料、核实情况、首付与代付关系。
- `完成`：客户经理手写签字、图片预览、下载图片和清空资料。

### 常用录入方式

#### 粘贴客户资料

右上角的“粘贴客户资料”图标可处理类似下面的文字：

```text
申请人姓名: 某客户
本人电话: 18800000000
本人目前工作单位名字: 示例信息技术有限公司
目前居住地址: 示例市示例区示例路 100 号
```

网页只识别以下四项，并在确认后写入表单：

- 客户姓名
- 客户电话
- 单位／公司名称
- 客户居住地址

原始粘贴文本不会保存；不在当前尽调表中的联系人、邮箱、婚姻状态等内容会被忽略。

#### 身份证识别

右上角的身份证图标支持从相册或文件中选择身份证正面图片。识别完成后，网页会展示结果供确认，再填入：

- 客户姓名
- 客户身份证号码
- 客户性别

请确保身份证文字方向正常、证件完整且清晰。图片识别是可选功能，需要完成下方的腾讯云配置。

## 表格规则

- 客户单位与自雇公司（如有）在网页中二选一，使用同一个“单位／公司名称”输入框。
- 生成图片时，名称只会写入所选的原始表格单元格，其他单元格保持空白。
- 客户居住地址、单位／公司名称和其他长内容会自动换行，不会超出表格边框。
- 客户经理签字使用手写签字板；“横屏全屏”会将整个签字界面和清除确认提示横向显示，可随时恢复且不会丢失笔迹；点击“保存签名”后签字板锁定，点击“清除签名”并确认后才能重签。
- 亲见日期由贷款页录入，并绘制在正式图片原主管签字的位置。


## 腾讯云身份证 OCR 配置（可选）

身份证 OCR 需要由云函数中转，不能把腾讯云密钥写进浏览器代码。

### 架构

```text
手机网页
  -> 腾讯云 Web 函数
  -> 腾讯云 IDCardOCR
  -> 返回姓名、身份证号、性别
```

### 云函数要求

- 使用 Node.js Web 函数。
- 使用腾讯云 `IDCardOCR` 接口。
- 函数 URL 允许你的网页域名访问，例如 `https://<GitHub 用户名>.github.io`。
- 浏览器图片提交到函数后，仅在请求内存中处理；函数不要保存身份证原图或在日志中输出图片内容。

### 建议的环境变量

以下变量只保存在云函数配置中，绝不能提交到 GitHub：

| 变量名 | 用途 |
|---|---|
| `ALLOWED_ORIGINS` | 允许调用函数的网页源，多个地址用英文逗号分隔。 |
| `OCR_ACCESS_KEY` | 网页首次识别时输入的私有密码。 |
| `OCR_TENCENT_SECRET_ID` | 专用于 OCR 的子用户 `SecretId`。 |
| `OCR_TENCENT_SECRET_KEY` | 专用于 OCR 的子用户 `SecretKey`。 |

为 OCR 子用户仅关联 `QcloudOCRFullAccess`，不要使用 `AdministratorAccess`。若密钥泄露，请立即在腾讯云 CAM 中禁用或删除该访问密钥并重新创建。

> 身份证图片和识别结果属于敏感个人信息。请只在获得客户授权、符合公司流程和适用法律要求的情况下使用该功能。
